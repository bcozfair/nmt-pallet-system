## 3.7 การออกแบบสิ่งนำออก (Output Design)

การออกแบบสิ่งนำออกของระบบ NMT Pallet System มุ่งเน้นการให้ข้อมูลสถานะคลังพาเลทที่แม่นยำและเข้าถึงได้ง่าย ทั้งในรูปแบบเอกสาร (Hard Copy) และการแสดงผลผ่านหน้าต่างโปรแกรม (Soft Copy) ดังนี้:

### 3.7.1 สำเนาถาวร (Hard Copy)

ผลผลิตในรูปแบบเอกสารหรือไฟล์ที่สามารถนำไปใช้งานภายนอกระบบได้ มีรายละเอียดดังนี้:

#### **1. ป้ายรหัสพาเลท (Pallet QR Code Label)**
*   **ข้อมูลที่แสดง**: รหัสพาเลท (Pallet ID) ในรูปแบบข้อความและรหัส QR Code สำหรับการสแกน
*   **รูปแบบการพิมพ์**: รองรับการพิมพ์รายชิ้นผ่านหน้าต่างจัดการพาเลท หรือการพิมพ์แบบกลุ่ม (Bulk Print) ในรูปแบบตารางเพื่อให้ประหยัดพื้นที่กระดาษ

#### **2. รายงานข้อมูลในรูปแบบ CSV (CSV Export Reports)**
ระบบรองรับการส่งออกข้อมูลดิบเพื่อการตรวจสอบและวิเคราะห์เชิงลึก โดยมีโครงสร้างคอลัมน์มาตรฐานดังนี้:
*   **Pallet ID**: รหัสอ้างอิงพาเลท (Primary Key)
*   **Status**: สถานะปัจจุบัน (available, in_use, damaged)
*   **Location**: สถานที่จัดเก็บหรือแผนกล่าสุดที่ถือครอง
*   **Date Added**: วันและเวลาที่ลงทะเบียนพาเลท (ISO Timestamp)
*   **Last Activity**: วันและเวลาที่มีการสแกนล่าสุด
*   **Last Action Type**: ประเภทรายการล่าสุด (check_out, check_in, report_damage, repair)
*   **Action By**: ชื่อ-นามสกุลของพนักงานผู้ทำรายการล่าสุด
*   **Last Checkout**: วันและเวลาที่เริ่มเบิกใช้งานล่าสุด
*   **Overdue Days**: จำนวนวันที่ใช้งานเกินกำหนด (คำนวณจาก Checkout Date เทียบกับปัจจุบัน)
*   **Evidence URL**: ลิงก์รูปภาพหลักฐานกรณีแจ้งพาเลทชำรุด

---

### 3.7.2 สำเนาชั่วคราว (Soft Copy)

#### **1. รายงานสถิติและกราฟ (Summary Chart)**
รวบรวมข้อมูลสถานะคลังแบบ Real-time เพื่อช่วยในการตัดสินใจเชิงบริหารจัดการ ประกอบด้วย 4 กราฟหลักดังนี้:

*   **กราฟสรุปสถานะพัสดุ (Status Donut Chart)**:
    *   **ลักษณะ**: กราฟวงกลม (Donut) แสดงสัดส่วนตามรหัสสถานะ
    *   **สีที่ใช้**: สีเขียว (Available), สีฟ้า (In Use), สีแดง (Damaged)
    *   **ฟังก์ชัน**: แสดงจำนวนรวม (Total Fleet) ที่กึ่งกลางกราฟ และปรับเปลี่ยนเป็นเปอร์เซ็นต์รายสถานะเมื่อมีการเลื่อนเมาส์ผ่าน (Hover)
*   **กราฟการถือครองรายแผนก (Location Inventory Chart)**:
    *   **ลักษณะ**: กราฟแท่งแนวตั้ง (Bar Chart) เรียงลำดับจากแผนกที่ถือครองพาเลทมากที่สุดไปหาน้อยที่สุด
    *   **ฟังก์ชัน**: กรองเฉพาะรายการที่ไม่อยู่ในคลังกลาง (Warehouse) เพื่อให้เห็นการกระจายตัวของทรัพย์สิน และรองรับการคลิกที่แท่งเพื่อกรองข้อมูลพาเลทในตารางหลัก
*   **กราฟแนวโน้มความเคลื่อนไหว (Activity Trend Chart)**:
    *   **ลักษณะ**: กราฟเส้น (Line Chart) แสดงแนวโน้มของกิจกรรมตามช่วงเวลา (วัน/สัปดาห์/เดือน)
    *   **ข้อมูลที่แสดง**: แยกสีตามเส้นธุรกรรม ได้แก่ การเบิกใช้งาน (Check-out), การคืนพัสดุ (Check-in), การพบความเสียหาย (Damage) และพัสดุรับใหม่ (New Acquisition)
*   **ตารางประเมินความเสี่ยงรายแผนก (Location Risk Matrix)**:
    *   **ลักษณะ**: รายการแบบการ์ดประเมินแผนกที่มีความเสี่ยงสูง (Risk Priority)
    *   **เกณฑ์การคำนวณ**: คำนวณ Risk Score จากจำนวนพาเลทที่ถือครองเกินกำหนด (Overdue - ส้ม) และพัสดุที่ชำรุดค้างในแผนก (Damaged - แดง) เพื่อแจ้งเตือนให้รีบดำเนินการติดตาม

#### **2. รายงานแจ้งเตือนผ่าน LINE (LINE Flex Messages)**
ระบบส่งการแจ้งเตือนอัตโนมัติ 2 ประเภทหลัก:
*   **รายงานตอนเช้า (Morning Overdue Alert)**:
    *   **Theme**: สีแดง (Warning)
    *   **Content**: จำนวนพาเลทที่ใช้งานเกินกำหนด (Overdue Count), เกณฑ์วันที่กำหนด (Threshold Days), และรายการแผนกที่ถือครองพาเลทเกินกำหนด (Location Breakdown)
*   **รายงานช่วงเย็น (Evening Daily Summary)**:
    *   **Theme**: สีน้ำเงิน (Informative)
    *   **Content**: ยอดพาเลททั้งหมด (Total Inventory), ตารางสรุปสถานะ (Status Grid), และรายชื่อสถานที่ที่มีความเคลื่อนไหว (Active Locations)

## 3.8 การออกแบบสิ่งนำเข้า (Input Design) และการออกแบบจอภาพ (Screen Design)

### 3.8.1 การออกแบบข้อมูลนำเข้า (Input Data Design)

หัวข้อนี้นำเสนอรายละเอียดของหน้าจอและฟอร์มที่มีการกรอกข้อมูล (Input Forms) ทั้งหมดในระบบ โดยแบ่งออกเป็นแต่ละหน้าจอดังนี้:

#### **1. ข้อมูลเข้าสู่ระบบ (Login Screen)**
*   **วัตถุประสงค์**: สำหรับการยืนยันตัวตนพนักงานเพื่อเข้าใช้งานระบบ
*   **Fields**: Employee ID (Required), Password (Min 6 chars)

#### **2. ข้อมูลรีเซ็ตรหัสผ่าน (Password Recovery Modal)**
*   **วัตถุประสงค์**: สำหรับขอรับลิงก์กู้คืนรหัสผ่านโดยการเลือกบัญชีผู้ดูแลระบบ
*   **Fields**: Select Admin Account (Dropdown List)

#### **3. ข้อมูลแจ้งพาเลทชำรุด (Damage Report Form)**
*   **วัตถุประสงค์**: บันทึกข้อมูลความเสียหายและแนบรูปภาพหลักฐานจากอุปกรณ์เคลื่อนที่
*   **Fields**: Pallet ID (Auto-scanned), Evidence Photo (Required), Remark (Optional)

#### **4. ข้อมูลการสแกนเบิก-คืนพาเลท**
*   **วัตถุประสงค์**: รับรหัสพาเลทผ่านการสแกน QR, ระบุแผนกปลายทาง และตรวจสอบรายการก่อนบันทึกธุรกรรม
*   **Fields**: 
    *   **QR Scanner View**: Scanned QR Content (Pallet ID)
    *   **Location Selection**: Department Selection (List Item Click)
    *   **Batch Scan List**: Review Scanned Pallet IDs & Confirmation Button

#### **5. ข้อมูลเพิ่มพาเลทใหม่ (Add New Pallet Modal)**
*   **วัตถุประสงค์**: ลงทะเบียนพาเลทใหม่เข้าสู่ระบบคลังพัสดุ
*   **Fields**: Pallet ID (Unique, Required), Initial Location (Dropdown Select)

#### **6. ข้อมูลแก้ไขข้อมูลพาเลท (Edit Pallet Modal)**
*   **วัตถุประสงค์**: แก้ไขรายละเอียดรหัสพาเลทหรือบันทึกหมายเหตุเพิ่มเติม
*   **Fields**: Updated Pallet ID, Pallet Remark (Edit Text)

#### **7. ข้อมูลแก้ไขรายการธุรกรรม (Edit Transaction Modal)**
*   **วัตถุประสงค์**: แก้ไขรายละเอียดของรายการเช็คอินหรือเช็คเอาท์ในกรณีที่บันทึกข้อมูลผิดพลาด
*   **Fields**: Transaction ID (Read-only), Adjusted Location (Dropdown Select), Edit Remark (Required)

#### **8. ข้อมูลจัดการบัญชีผู้ใช้งาน (Add/Edit User Modal)**
*   **วัตถุประสงค์**: เพิ่มหรือแก้ไขข้อมูลส่วนบุคคลและระดับสิทธิ์ของพนักงาน
*   **Fields**: Employee ID (Unique), Full Name, Department, Role (Staff/Admin), Password

#### **9. ข้อมูลจัดการสถานที่ (Add/Edit Location Modal)**
*   **วัตถุประสงค์**: เพิ่มหรือแก้ไขรายชื่อแผนกและจุดจัดเก็บหลักในระบบ
*   **Fields**: Location Name (Unique), Active Status (Toggle)

#### **10. ข้อมูลการตั้งค่าระบบ (System Settings Form)**
*   **วัตถุประสงค์**: กำหนดค่าการเชื่อมต่อ API และเงื่อนไขการทำงานอัตโนมัติของระบบ
*   **Fields**: LINE Channel Token, LINE Target ID, Overdue Threshold (Days Count)

### 3.8.2 การออกแบบจอภาพ (Screen Design)

หัวข้อนี้นำเสนอหน้าจอแสดงผลข้อมูลทั่วไป (General Views) และส่วนประกอบในการโต้ตอบหลัก:

#### **ส่วนของผู้ใช้งานระดับพนักงาน (Staff Mobile Interface)**
1.  **หน้าจอหลัก (Mobile Home)**: แสดงข้อมูลเบื้องต้นและเมนูสำหรับการปฏิบัติงาน
2.  **สัญลักษณ์ตอบสนองการสแกน (Feedback Overlay)**: การแสดงผล Visual Feedback สีเขียว/แดง เมื่อสแกนสำเร็จหรือล้มเหลว
3.  **หน้าจอประวัติการใช้งาน (Activity History)**: แสดงรายการความเคลื่อนไหวส่วนบุคคลในรูปแบบ Timeline พร้อมระบบค้นหาและกรองข้อมูลตามวันที่

#### **ส่วนของผู้ดูแลระบบ (Admin Web Interface)**
1.  **แดชบอร์ดสรุปภาพรวม (Admin Dashboard)**: หน้ารวมรายงานสถิติและกราฟสรุปสถานะทั้งหมด
2.  **หน้าบริหารจัดการคลัง (Inventory Management)**: หน้าหลักสำหรับดูรายการ ค้นหา และจัดการสถานะพาเลท
3.  **หน้าแสดงรายละเอียดพาเลท (Pallet Detail)**: แสดงข้อมูลเจาะลึกของพาเลทแต่ละรายการ รวมถึงประวัติความเคลื่อนไหวย้อนหลัง (Pallet Activity History)
4.  **หน้าจัดการผู้ใช้งาน (User Management View)**: ตารางรายชื่อและสถานะของพนักงานในระบบ
5.  **หน้าจัดการสถานที่ (Location Management View)**: ตารางแสดงรายชื่อแผนกและจุดเก็บพาเลทที่เปิดใช้งาน
6.  **หน้าจัดการธุรกรรม (Transaction Management)**: รายการธุรกรรมย้อนหลังทั้งหมดของระบบ (Audit Trail) พร้อมระบบตรวจสอบและแก้ไขข้อมูล
7.  **หน้าต่างพิมพ์รหัสกลุ่ม (QR Code Master Sheet)**: หน้าต่างพรีวิวรหัส QR สำหรับพาเลทที่เลือก เพื่อใช้ในการพิมพ์ (Print) หรือบันทึกเป็นไฟล์ภาพ (Save Image)
